# Сборник задач

Сборник задач по курсу "Прикладное программирование" (обработка данных).


## Установка

Скопируйте репозиторий в папку на локальном компьютере. Лучше с помощью git:
```
git clone https://github.com/AltmanEA/course_data_lab
```
Перейдите в папку и установите необходимые модули
```
cd course_data_lab
npm install
```
Для проверки решения задачи запустите соответствующий тест, например:
```
npm run test mongo01
```
Для прохождения контрольных установите в vscode расширение ```examenator-X.X.X.vsix``` (можно просто перетащить этот файл на вкладку "Extentions").

### Задачи, требующие сервера.

При решении задач из блоков mongo и prisma необходимо запустить серверы с помощью ```docker-compose up``` (см. ```docker-compose.yml```). 

Проверьте настройки подключения к серверам в файлах ```.env``` и ```src/mongo/mongo_init.ts```, по умолчанию строки подключения настроены на работу nodejs в контейнере. При использовании nodejs на ОС замените ```mongo``` и ```postgres``` на ```localhost```.

Для ```prisma``` нужно создать таблицы в базе данных:
```
npx prisma migrate reset
```

## Рекомендации по решению

Решать задачи нужно с помощью тех средств языка программирования, которые изучаются в текущем блоке. Например, если изучаются функции для работы с массивами, то не стоит использовать циклы; если работаете с базой данных, то не нужно загружать всю базу данных в память и решать задачу с помощью функций для работы с массивами. Хотя это и не спортивное программирование и оптимизация решений не требуется, тем не менее не стоит приводить решение в лоб. При проведении контрольных работ корректность решения будет выборочно проверятся вручную.

Решать задачи рекомендуется самостоятельно. Смысл заданий - набить руку на решении подобных задач. Любой ИИ сможет легко решить эти задачи (не всегда корректно в терминах предыдущего абзаца), но просмотр чужого решения не сформирует нужные нейронные связи для естественного интеллекта.
